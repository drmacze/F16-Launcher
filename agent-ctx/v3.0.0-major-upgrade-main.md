### Task 1: Fix Telemetry (app_events kosong) ✅
**File**: `Telemetry.kt`
**Problem**: `app_events` table empty — telemetry not reaching Supabase. Errors swallowed silently, no way to debug.
**Root causes**:
1. RLS policy blocked anon INSERT (fixed by SQL v14)
2. Errors swallowed silently with no logging
3. Response code not checked

**Fix**:
- Added `Log.w(TAG, ...)` on all failures (filter: `adb logcat -s DLavieTelemetry`)
- Added `Log.i(TAG, ...)` on success (verbose — visible with `-v`)
- Logs HTTP code + 200-char body excerpt on non-2xx responses
- Auth handling: logged-in → user JWT; logged-out → anon key (RLS `app_events anon insert` from SQL v14)
- `user_id` omitted when not logged in (column is nullable)

### Task 3: Fix Badge Awarding (user_badges kosong) ✅
**File**: `CommunityApi.java`
**Problem**: `user_badges` table empty — badges never awarded.

**Old implementation** (removed):
- Client-side loop: fetch all badge definitions → fetch user_badges → for each badge, fetch related data (sessions, posts, follows, etc.) → check requirement → INSERT
- N+1 queries per badge check (very inefficient)
- RLS policy blocked INSERT (fixed by SQL v14: `user_badges own insert` with `auth.uid() = user_id`)

**New implementation**:
```java
public Object checkAndAwardBadges() throws Exception {
    JSONObject body = new JSONObject().put("p_user_id", userId());
    String resp = request("POST", "/rest/v1/rpc/check_and_award_badges", body, true, null);
    // Parse jsonb array response (newly-awarded badge codes)
    ...
}
```
- Single RPC call (was N+1)
- All requirement logic lives in SQL `security definer` function (atomic, consistent)
- Returns `JSONArray` of newly-awarded badge codes (logged via `Log.i`)
- Errors swallowed with `Log.w` (badges not critical — don't crash caller)

**Call sites** (auto-triggered from API methods):
- `login()` → logActivity("login") + checkAndAwardBadges()
- `register()` → logActivity("register") + checkAndAwardBadges()
- `createFeedPost()` (non-draft) → logActivity("create_post") + checkAndAwardBadges()
- `addComment()` → logActivity("create_comment") + checkAndAwardBadges()
- `followUser()` → logActivity("follow") + checkAndAwardBadges()
- `submitRating()` → logActivity("rate_game") + checkAndAwardBadges()
- `recordGameSession()` → logActivity("game_session") (badge check called separately in ON_RESUME)
- `launchGame()` (Kotlin) → logActivity("game_launch")

## Errors Encountered + Fixes

### Error 1: Kotlin type inference failure in DLavieGuidedActivity.kt
**Error**:
```
e: DLavieGuidedActivity.kt:798:231 Type mismatch: inferred type is Any but () -> Unit was expected
e: DLavieGuidedActivity.kt:824:159 Type mismatch: inferred type is Any but () -> Unit was expected
```
**Root cause**: Pre-existing bug — `if (cond) { guidedLaunchGame(context) } else openData` evaluates to `Any` (mixing `Unit` and `() -> Unit`), but `GuidedPrimaryCta` expects `() -> Unit`. Was likely never caught because previous builds used `assembleDebug` or skipped release compilation.
**Fix**: Wrapped the call in a lambda: `if (cond) { { guidedLaunchGame(context) } } else openData`. Now both branches are `() -> Unit`.

### Error 2: `jlink` missing (JDK 21 headless)
**Error**:
```
jlink executable /usr/lib/jvm/java-21-openjdk-amd64/bin/jlink does not exist.
```
**Root cause**: System had only `openjdk-21-jre-headless` (no `jlink`). AGP uses `jlink` to build JDK image for compilation.
**Fix**: Downloaded `openjdk-21-jdk-headless` deb, extracted to `$HOME/jdk21/`, then copied missing `lib/*.so`, `conf/`, `bin/java` from system JRE to make a complete JDK at `$HOME/jdk21/usr/lib/jvm/java-21-openjdk-amd64/`. Set `JAVA_HOME` to this path.
